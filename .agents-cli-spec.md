# Agent Spec: FormGuide

## Overview
FormGuide is a conversational assistant that helps UK residents prepare a Universal Credit application. The agent gathers required information across several sections (housing, income/savings, childcare, and health), validates it to ensure completeness and correctness, and produces a structured plain-English summary document that the user can use for their actual application.

## Example Use Cases
### Case 1: Information Gathering
- **User input**: "I need help with my Universal Credit application."
- **Agent response**: Welcomes the user, explains the process, and starts by asking about their housing situation (e.g., rent, mortgage, local authority).
- **Subsequent steps**: Prompts the user through income/savings, childcare, and health questions, validating the answers.

### Case 2: Generating the Summary
- **User input**: After answering all sections.
- **Agent response**: Generates a structured plain-English summary document of the gathered housing, income, childcare, and health details.

## Architecture
The agent must be form-agnostic at the orchestrator level. All Universal-Credit-specific question logic, thresholds, and rules belong in a dedicated skill (`.agents/skills/universal-credit/SKILL.md`), not hardcoded into `app/agent.py`. The orchestrator's job is generic:
1. Run a conversational section-by-section flow.
2. Call the active form's skill for domain logic and question schemas.
3. Validate inputs using provided tools and rules.
4. Summarize gathered details.
5. Obtain user confirmation.
6. Generate output.
This design supports adding more UK government forms as new skills in the future without modifying the core agent.

## Tools Required
- **Postcode Validation Tool**: A custom Model Context Protocol (MCP) tool that validates any UK postcode provided by the user using the postcodes.io API (`https://postcodes.io`). UK postcodes must not be accepted as unverified free text.
- **Auth**: Plain Gemini API key via `GEMINI_API_KEY` (no Vertex AI/GCP auth).

## Constraints & Safety Rules
- **No official legal/financial advice**: The agent must clearly state that it is an assistant to help gather information, not a legal advisor or DWP representative.
- **Data Privacy**: The agent must not store or persist sensitive personal data (e.g., National Insurance numbers, bank details) outside the immediate session context.

## Success Criteria
- Guided Universal Credit flow: Guides the user through all four sections: housing, income/savings, childcare, and health.
- Deterministic threshold checks: The £16,000 savings eligibility threshold must be checked in deterministic Python code, never inferred by the LLM.
- Explicit confirmation: Before generating the final summary, the agent must restate all gathered information back to the user and obtain explicit confirmation before producing the output document.
- Performs validation on details (e.g., UK postcode validation via postcodes.io MCP tool).
